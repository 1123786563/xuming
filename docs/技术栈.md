1. 核心开发框架：Flutter (Dart)
推荐原因：Flutter 的 Skia 渲染引擎能够实现像素级的 UI 还原，对于“发光”、“渐变”、“磨砂玻璃”以及“自定义着色器（Shaders）”的支持远超 React Native。

状态管理：Riverpod。用于管理 HP 血条实时同步、续命币余额更新等全局状态。
导航与路由：GoRouter。支持复杂的页面跳转逻辑，方便实现从任何页面一键切回“Boss Mode”。
2. 3D 指引与动画实现：Three.js / Rive / SceneKit
挑战：App 需要展示写实 3D 人体模型及其热力图，同时要保持轻量。

方案 A (推荐)：Rive (for Flutter)。用于实现 2D/2.5D 的复杂机械结构动画。它非常轻量，且支持状态机交互（比如根据压力大小实时改变模型颜色）。
方案 B：Flutter Scene (Experimental) 或 WebView 嵌入 Three.js。如果必须使用写实 3D（GLB 格式），建议通过 Webview 嵌入优化后的 Three.js 页面，或者使用 Flutter 的 three_dart 库。
3. 视觉特效实现 (Visual FX)
故障艺术 (Glitch)：通过 Flutter Custom Shaders (Fragment Shaders)。编写 .frag 脚本实现像素偏移、RGB 分离和噪点效果。这是性能损耗最低且效果最硬核的方案。
发光 (Neon Glow)：利用 BoxShadow 的多层叠加和 MaskFilter.blur。
ASMR 音效：Audioplayers 库。支持多声道同时播放（背景 ASMR 氛围音 + 动作复位“咔哒”声）。
4. 底层交互与传感器：Native Bridge (Method Channels)
Boss Mode 触发：使用 Sensors_plus 监听加速度计。当检测到快速晃动（摇一摇）或桌面翻转时，通过原生代码瞬间置顶伪装界面。
桌面小组件：Home_widget 插件。它能将 Flutter 逻辑桥接到原生 iOS (SwiftUI) 和 Android (AppWidget) 的组件开发中。
触觉反馈：Haptic_feedback。调用系统的 Taptic Engine 实现不同频率的震动反馈。
5. 后端与数据同步：Supabase (Serverless)
推荐原因：它是一个开源的 Firebase 替代方案，基于 PostgreSQL，非常适合处理实时数据（如：同事之间的“投弹”互动）。

Auth (认证)：内置手机号验证码及第三方登录。
Realtime：用于实现“生存排行榜”的实时更新以及同事间的消息推送。
Storage：存储用户生成的“续命保单”海报。
6. 基础设施与自动化
CI/CD：Codemagic。专为 Flutter 设计的自动化构建工具，快速分发测试版给开发者。
错误监控：Sentry。实时捕获应用崩溃，尤其是复杂的 Shader 渲染错误。
总结：技术架构图
[ 用户端 (Flutter) ] 
   |-- UI 层: Shaders (.frag) + Rive (Animations)
   |-- 逻辑层: Riverpod (State) + Method Channels (Haptics/Sensors)
   |-- 存储层: Isar (Local DB for logs)
   |
[ 桥接层 (Native Swift/Kotlin) ]
   |-- iOS SwiftUI Widgets / Android Widgets
   |-- Boss Mode Overlay System
   |
[ 服务端 (Supabase) ]
   |-- PostgreSQL (UserData/Coins)
   |-- Edge Functions (Weekly Report Generation)
   |-- Realtime (Social Bombs)
这套技术栈能够确保应用在 Android 和 iOS 上都表现得像一个真正的“次世代生物监控终端”。
